# MarksAgent Ecosystem

MarksAgent — это событийно-ориентированная мультиагентная система для оценки когнитивных стратегий студентов при верификации ИИ-контента. Проект предназначен для выявления паттернов фактчекинга на основе того, как студенты анализируют и размечают текст.

## Состав системы (Архитектура)

Проект имеет современную микросервисную архитектуру, где изолированные узлы общаются друг с другом асинхронно через брокер сообщений Redis.

1. **Frontend UI (Next.js / React / TailwindCSS)**
   Интерактивный веб-интерфейс (холст), где пользователь читает текст и использует маркеры: зеленый (правда), желтый (сомнение) и красный (ложь). Фронтенд поддерживает WebSocket соединение для получения результатов работы агентов в реальном времени.

2. **API Gateway (FastAPI)**
   Шлюз, принимающий JSON-запросы от сайта. Публикует задачи в очередь `marks_agent_queue` (в Redis) и держит открытым WebSocket-сервер для мгновенной рассылки ответов обратно в браузер.

3. **MarksAgent (Аналитическое/Математическое ядро)**
   Агент, работающий по строгим математическим правилам `_calculate_metrics`. Рассчитывает Precision, Recall, F1-Score на основе эталонной базы ошибок (Ground Truth) и классифицирует пользователя (например, "точный детектив" или "осторожный сомневающийся"). Результаты записываются в локальную БД SQLite (`data/students_profiles.db`).

4. **TutorAgent (LLM AI-Ментор)**
   Агент, который прослушивает вердикты `MarksAgent` и обращается к локальной нейросети (Ollama, модель llama3:8b). На основе метрик ИИ генерирует персонализированный ободряющий фидбек с советами по улучшению навыков проверки фактов.

## Требования для запуска

Для запуска проекта локально на вашей машине необходимо установить:
- **Docker** и **Docker Compose**
- **Node.js** (версия 18 и новее)
- **Ollama** (для запуска локальной языковой модели)

---

## Пошаговое руководство по запуску

Система спроектирована так, чтобы разворачиваться минимальным количеством команд.

### Шаг 1: Подготовка AI-модели
Убедитесь, что у вас установлена программа Ollama. Скачайте и запустите в фоне языковую модель (по умолчанию используется llama3 с 8 миллиардами параметров). 
Откройте первый терминал и выполните:
```bash
ollama run llama3:8b
```
Дождитесь скачивания и инициализации модели (в терминале появится строка для общения). Приложение Ollama продолжит слушать запросы агента по порту `11434`.

### Шаг 2: Настройка окружения
В корне проекта находится файл конфигурации `.env.example`. Скопируйте его, чтобы создать рабочий конфиг:
```bash
cp .env.example .env
```
Вы можете отредактировать файл `.env`, если используете другую модель (например, `mistral`).

### Шаг 3: Запуск Бэкенд-кластера (Docker)
Убедитесь, что Docker Desktop запущен. Откройте второй терминал в корневой папке проекта и запустите сборку всех микросервисов:
```bash
docker-compose up --build
```
Docker автоматически:
- Поднимет Redis (брокер сообщений).
- Поднимет контейнер с FastAPI (`api_gateway.py`) на порте `8000`.
- Поднимет фоновых асинхронных обработчиков `MarksAgent` и `TutorAgent`.
- Настроит безопасные пулы для SQLite и проброс сети к Ollama через `host.docker.internal`.

### Шаг 4: Запуск Пользовательского Интерфейса (Frontend)
Откройте третий терминал, перейдите в папку с кодом сайта и запустите сервер для разработки:
```bash
cd frontend
npm install
npm run dev
```

### Шаг 5: Использование
Откройте в браузере адрес: `http://localhost:3000`
1. Выберите в панели управления нужный цвет маркера.
2. Кликните по предложению, чтобы окрасить его.
3. Нажмите кнопку "Отправить на проверку".
4. Дашборд справа мгновенно отобразит ваши математические метрики (через WebSocket), а спустя несколько секунд подтянется текстовый комментарий от искусственного интеллекта.

## Техническое обслуживание и Тесты

Ядро расчета метрик покрыто автоматическими юнит-тестами. Данные тестируют математический расчет Precision, Recall, и F1 Score, а также логику распределения пользователей по 5 когнитивным паттернам. Для проверки консистентности логики агентов запустите:
```bash
pip install -r requirements.txt
pytest test_marks_agent.py
```
Система не использует `print()` для логирования — все процессы протоколируются модулем `logging` в консолях Docker-контейнеров. Проект полностью очищен от технического долга, имеет систему Graceful Shutdown и готов к масштабированию.
