# MarksAgent

**MarksAgent** — это интеллектуальный узел в мультиагентной системе для анализа когнитивных стратегий верификации ИИ-контента. Агент принимает сырые данные о цветовых пометках, оставленных студентами при анализе текста, и превращает их в когнитивные метрики.

Он работает на основе символьного ИИ (дерево решений), обеспечивая стопроцентную прозрачность алгоритмов и высокую скорость обработки данных.

-----

## Подготовка: что нужно установить перед началом

Чтобы запустить проект на вашем компьютере, вам понадобятся три базовых инструмента:

1.  **Python** (версии 3.9 или выше) — язык программирования, на котором написан проект. [Скачать Python](https://www.python.org/downloads/)
2.  **Git** — система контроля версий для скачивания проекта. [Скачать Git](https://git-scm.com/downloads)
3.  **Docker** (рекомендуется) или **Redis** — база данных (брокер сообщений), через которую агенты обмениваются информацией. [Скачать Docker Desktop](https://www.docker.com/products/docker-desktop/)

-----

## Пошаговое руководство по запуску

Следуйте этой инструкции строго по порядку. Для работы вам понадобится открыть **Терминал** (в macOS/Linux) или **Командную строку / PowerShell** (в Windows).

### Шаг 1. Скачивание (клонирование) проекта

Откройте терминал, перейдите в папку, где хотите сохранить проект, и выполните команду:

```bash
git clone https://github.com/epocson/MarksAgent.git
cd MarksAgent
```

### Шаг 2. Создание виртуального окружения

Виртуальное окружение изолирует этот проект от других файлов на вашем компьютере, чтобы избежать конфликта версий.
Находясь в папке `MarksAgent`, введите:

**Для Windows:**

```bash
python -m venv venv
venv\Scripts\activate
```

**Для macOS и Linux:**

```bash
python3 -m venv venv
source venv/bin/activate
```

*(Если всё прошло успешно, в начале строки терминала появится надпись `(venv)`).*

### Шаг 3. Установка необходимых библиотек

Теперь, когда виртуальное окружение активно, установите все зависимости проекта:

```bash
pip install -r requirements.txt
```

### Шаг 4. Запуск базы данных Redis

Агенту требуется сервер Redis для работы с очередями сообщений. Выберите способ, который подходит для вашей операционной системы. Эту вкладку терминала/консоли **закрывать нельзя**!

**Универсальный и самый простой способ (через Docker для любой ОС):**
Если у вас запущен Docker Desktop, просто выполните в новом окне терминала:

```bash
docker run -d -p 6379:6379 redis
```

**Альтернатива для macOS (без Docker):**
Если у вас установлен пакетный менеджер Homebrew, выполните:

```bash
brew install redis
/opt/homebrew/opt/redis/bin/redis-server /opt/homebrew/etc/redis.conf
```

### Шаг 5. Запуск основного агента (MarksAgent)

Откройте **вторую (новую) вкладку терминала** (не забудьте снова перейти в папку проекта `cd MarksAgent` и активировать окружение `venv\Scripts\activate` или `source venv/bin/activate`).

Запустите скрипт агента, который начнет слушать очередь сообщений:

```bash
python marks_agent.py
```

*В консоли должно появиться сообщение `Started listening to marks_agent_queue...`. Не закрывайте эту вкладку — агент работает в фоновом режиме и ждет данные.*

### Шаг 6. Использование системы и тестирование

Теперь ядро системы активно. Откройте **третью вкладку терминала** (также перейдите в папку проекта и активируйте окружение).

**1. Проверка работоспособности (Автоматические тесты)**
Чтобы убедиться, что вся математика и логика агента работают без ошибок:

```bash
pytest test_marks_agent.py -v
```

*(Если в проекте также присутствует `test_agent.py`, вы можете запустить и его командой `pytest test_agent.py -v`)*

**2. Запуск симуляции (если файлы присутствуют в проекте)**
Если вы используете скрипт-эмулятор для генерации потока тестовых студентов:

```bash
python simulate_ecosystem.py
```

**3. Запуск визуального дашборда (если файл присутствует в проекте)**
Интерактивный интерфейс для преподавателя (откроется в браузере):

```bash
streamlit run dashboard.py
```

## Структура проекта

Для тех, кто хочет разобраться, как всё устроено внутри:

  * `marks_agent.py` — "мозг" системы. Слушает очередь Redis, проверяет входящие JSON-данные, считает когнитивные метрики, классифицирует профиль студента и записывает результаты в локальную базу данных SQLite (`students_profiles.db`).
  * `test_marks_agent.py` и `test_agent.py` — файлы с автоматическими тестами. Проверяют правильность расчетов и обработки ошибок.
  * `requirements.txt` — список библиотек Python, необходимых для работы проекта.
  * `.agents/rules` — директория, содержащая внутренние правила/промпты для агентов (если применимо).
  * `simulate_ecosystem.py` *(опционально)* — генератор фейковых студентов для проверки нагрузок системы.
  * `dashboard.py` *(опционально)* — веб-интерфейс на Streamlit для наглядного отображения результатов и XAI-объяснений.
