# MarksAgent

MarksAgent — это интеллектуальный узел в мультиагентной системе для анализа когнитивных стратегий верификации ИИ-контента.

## Описание агента
MarksAgent принимает сырые данные о цветовых пометках, оставленных студентами при анализе текста (зелёные — уверенность, жёлтые — сомнение, красные — найденные ошибки). Агент превращает эти сырые данные в числовые признаки (доли пометок, индексы сомнения, базовые счетчики) и объясняет паттерны внимания студентов (например, «точный детектив» или «пассивный наблюдатель»). 

## API агента (Redis Pub/Sub)
Агент подписывается на очередь `marks_agent_queue` и слушает JSON-сообщения.

### Пример входящего JSON-сообщения
```json
{
  "total_fragments": 10,
  "marks": {
    "green": [1, 2],
    "yellow": [3],
    "red": [4]
  },
  "ground_truth_errors": [4, 5]
}
```

### Пример исходящего JSON-сообщения
```json
{
  "features": {
    "green_ratio": 0.2,
    "yellow_ratio": 0.1,
    "red_ratio": 0.1,
    "yellow_green_ratio": 0.5,
    "uncertainty_index": 0.5
  },
  "cognitive_pattern": "смешанный",
  "xai_explanations": {
    "green": "Студент уверен в правильности выделенных фрагментов.",
    "yellow": "Зоны сомнения — индикатор повышенной когнитивной нагрузки.",
    "red": "Фрагменты идентифицированы как содержащие концептуальные или фактические ошибки.",
    "summary": "Выявлен паттерн «смешанный» на основе анализа цветовых выделений."
  },
  "metrics": {
    "true_positives": 1,
    "false_positives": 0,
    "false_negatives": 1,
    "precision": 1.0,
    "recall": 0.5,
    "f1_score": 0.667
  }
}
```

## Инструкция по локальному запуску

1. **Запустите Redis через Docker:**
   ```bash
   docker run -d -p 6379:6379 redis
   ```

2. **Установите зависимости:**
   ```bash
   pip install -r requirements.txt
   ```

3. **Запустите агента:**
   ```bash
   python marks_agent.py
   ```

## Запуск автотестов
Для запуска тестов используется фреймворк `pytest`. Выполните следующую команду в корне проекта:
```bash
pytest test_marks_agent.py -v
```
