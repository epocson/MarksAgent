# MarksAgent

MarksAgent — это интеллектуальный узел в мультиагентной системе для анализа когнитивных стратегий верификации ИИ-контента. Агент принимает сырые данные о цветовых пометках, оставленных студентами при анализе текста, и превращает их в когнитивные метрики. 

Он работает на основе символьного ИИ (дерево решений), обеспечивая стопроцентную прозрачность и высочайшую скорость обработки.

## Структура решения

* `marks_agent.py` — основной файл агента. Слушает очередь Redis, валидирует JSON через Pydantic, рассчитывает метрики (доли от текста и от всех пометок), классифицирует студента и сохраняет данные в локальную БД SQLite (`students_profiles.db`).
* `simulate_ecosystem.py` — скрипт-эмулятор. Генерирует тестовых студентов, отправляет их данные в очередь и перехватывает итоговые результаты, выводя их в консоль в виде таблицы.
* `dashboard.py` — интерактивный визуальный интерфейс на базе Streamlit для преподавателя и исследователя. Позволяет вручную менять параметры студента и мгновенно видеть диагноз системы и XAI-объяснения.
* `test_marks_agent.py` — скрипт автоматизированного тестирования логики и математических метрик агента с использованием фреймворка pytest.

## Инструкция по локальному запуску

Для запуска всей системы необходимо открыть три разные вкладки в терминале.

### Шаг 1. Запуск сервера базы данных Redis
Вам необходим работающий сервер Redis, через который агенты обмениваются сообщениями. В первой вкладке выполните команду запуска базы данных и не закрывайте ее.

**Для macOS:**
```bash
/opt/homebrew/opt/redis/bin/redis-server /opt/homebrew/etc/redis.conf
```
*(В качестве альтернативы можно использовать `docker run -d -p 6379:6379 redis`)*

**Для Windows:**
Поскольку сервер Redis официально не поддерживает установку напрямую на платформу Windows, самым надежным и быстрым вариантом является использование **Docker**:
```bash
docker run -d -p 6379:6379 redis
```
*(Также возможен вариант установки через Windows Subsystem for Linux - WSL2)*

### Шаг 2. Запуск основного агента (MarksAgent)
Откройте вторую вкладку, перейдите в папку с проектом и запустите скрипт агента. Он начнет слушать очередь сообщений.

```bash
python marks_agent.py
```

В консоли появится сообщение: `Started listening to marks_agent_queue...`. Не закрывайте эту вкладку.

### Шаг 3. Запуск симуляции или дашборда
Теперь, когда ядро системы готово, вы можете взаимодействовать с ним из третьей вкладки терминала.

Для эмуляции потока студентов выполните:
```bash
python simulate_ecosystem.py
```

Для запуска интерактивного визуального дашборда преподавателя выполните:
```bash
streamlit run dashboard.py
```

## Требования и установка зависимостей
Если вы запускаете проект впервые, убедитесь, что установлены необходимые библиотеки Python:
```bash
pip install -r requirements.txt
```
А также запустите тесты, чтобы удостовериться, что логика работает корректно:
```bash
pytest test_marks_agent.py -v
```
